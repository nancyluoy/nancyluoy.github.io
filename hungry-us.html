Hungry US Redesign

JAN - MAR 2019

Role

Research, Ideation, UX, UI, Prototyping

Tool

Invision Studio

Team

Yi Luo, Alicia Ponce, Siyu Hu, Nha Vuong, Ziyu Song, Richard Liu

Project Overview

HungryUS is a food delivery app that provides free lunch and dinner delivery services to specific locations in San Diego, Irvine, and Los Angeles. It allows customers to order food from multiple restaurants and have them delivered at the same time without paying any delivery or service fee. Because of the way HungryUS is operated, instead of allowing customers to order anytime and delivering food to customers’ door, customers will need to order in a given time period and pick up their food at the location of their choice.

Challenge

The project is part of my HCI project course. The goal of this project is to redesign HungryUS's iOS mobile application interface to provide a better user experience. During the process, we conducted multiple research to thoroughly evaluate the app and identify the problems, and redesigned the application based on our findings to make it more compatible with HungryUS's unique operating model.

User Research

Methods

Intervew,   Surveys,   Affinity Diagram

Our user population is made up primarily of cost-sensitive customers looking for the most convenient way to get food. We conducted some user research to study their behavior and routines on using food delivery apps. According to our research, the top 3 factors that users care about the most are:

Cost
Speed
Pictures and Ratings

Usability Issues

We performed Cognitive Walkthrough and Usability Testing to identify usability issues that the current system has. There are 4 major painpoints:

Providing Inoperative Functionalities

The "Nite Snack" and "Groupon" feature were not available. There is no announcement anywhere on the system to notify users that these features are currently unavailable until users clicks on them.

Important feature not discoverable

In the "Meal Bus Special" section, there is nothing to indicate how to sort or filter the menu. In fact, the filter feature would only display when users click the search bar on the top of the home page.

Confusing visual design

Participants indicated that some icons are too small for them to notice at their first glance. There is no obvious path to lunch or dinner menu, and the cut-off time for the next meal is not visible either. They also expressed that the homepage had a lot of information that made it difficult to understand and navigate.

Uninformative Feedback

When users try to add something to the shopping cart, the cart icon will show the total number of items added to the cart as shown in the picture. However, the icon itself is so small that it is hard to see the change, which makes users confused if the item was added successfully.

Redesign

Target 1: Standardize the Homepage

We removed the inoperative features and reorganized the layout of the homepage. Instead of a horizontal scrolling list, we decided to display dishes in the "What's New" section using carousel to make the pictures bigger and more appealing. We use the corresponding menu color (yellow for lunch, purple for dinner) as the background of the current item featured to signify which menu the dish was from. We also redesigned the "Meal Bus Special" section to give a more pronounced distinction between the lunch and dinner menu. In addition, we moved the cart to the top-left corner so it is noticeable without interfering the menu items.

Target 2: Feedback for Adding items

The centered pop-up window serves as the secondary confirmation when adding an item, before showing the shopping cart has changed on the main page, the additional message can make sure that user has been notified with his/her current state, reducing the confusion of not sure if the selection has been added or not.

Target 3: Access to Sort/Filter Features

We added a sort/filter feature to the main menu and rewrote the tags to avoid unnecessary confusion. We changed the sorting tag to “New,” “By Rating,” and “By Price.” According to our results in the usability tests, these are the three top priorities when users are searching. We add the changes for arrows next to “By Rating” and “By Price,” so users can know whether they are sorting from high to low or vice versa.

Target 4: Emphasize Order Cut-off Times

The new design features two main changes: the daily meal status at the top, and the countdown at the bottom. They are fixed in position so users won't accidentally scroll by and miss the cut-off time.
